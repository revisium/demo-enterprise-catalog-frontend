# Explainer Widget

The explainer widget is a secondary inspection surface. It must not replace the
catalog experience or dominate the first page read.

## Mock-First State

During frontend-first work, the widget names future evidence:

- candidate Revisium project/table/row ownership;
- nested objects and arrays shown by the page;
- future files, formulas, branches, revisions, or backend interactions.

## Connected State

After backend/Revisium integration, the widget should only show real evidence:

- GraphQL request and variables;
- sample response;
- Revisium project/table/row links;
- REST or MCP equivalents when available;
- revision or diff links when available.
