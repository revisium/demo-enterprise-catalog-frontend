# Bootstrap Notes

This repo was bootstrapped from the `demo-rpg-frontend` operating model, but the
development order is different.

## Enterprise Catalog Order

1. Document the most important pages.
2. Set up SSR stack, CI, review rules, and Sonar expectations.
3. Build several frontend iterations with typed mock data.
4. Stabilize design system, responsive behavior, and proof widgets.
5. Convert mocks into Revisium/backend contracts.
6. Implement backend and real Revisium connections.

Use this order for future enterprise catalog demos where UI clarity should drive
the first contracts.
